# How to Build Custom Email Templates

This guide shows how to override the default email templates provided by `@nik0di3m/medusa-plugin-smtp-mailing` with your own branding and content.

## Overview

There are two ways to customize emails:

| Approach | What it does | Best for |
|---|---|---|
| `templateOverrides` | Replace specific email templates entirely | Changing content, translations, per-template design |
| `baseLayoutOverride` | Replace the HTML layout wrapper for ALL default templates | Rebranding (logo, colors, footer) without rewriting every template |

Both are configured via provider options in `medusa-config.ts`. You can combine them.

---

## 1. Override a specific template

### Step 1: Create the template file

```ts
// src/email-templates/order-placed.ts
import type { EmailTemplate } from "@nik0di3m/medusa-plugin-smtp-mailing/types"

// Optional: use the plugin's built-in layout wrapper
import { baseLayout } from "@nik0di3m/medusa-plugin-smtp-mailing/templates"

export const subject: EmailTemplate["subject"] = (data) => {
  return `Order Confirmation #${data.display_id} - MyStore`
}

export const html: EmailTemplate["html"] = (data) => {
  const items = (data.items || [])
    .map(
      (item: any) =>
        `<tr>
          <td>${item.title || item.product_title || "Item"}</td>
          <td>${item.quantity || 1}</td>
        </tr>`
    )
    .join("")

  const content = `
    <h2>Thanks for your order!</h2>
    <p>Hi ${data.customer_name || "Customer"},</p>
    <p>Your order <strong>#${data.display_id}</strong> is being processed.</p>
    <table>
      <thead><tr><th>Item</th><th>Qty</th></tr></thead>
      <tbody>${items}</tbody>
    </table>
  `

  // Use the plugin's default layout, or write your own HTML from scratch
  return baseLayout(content, "MyStore")
}
```

### Step 2: Register in medusa-config.ts

```ts
// medusa-config.ts
import { defineConfig } from "@medusajs/framework/utils"
import * as orderPlaced from "./src/email-templates/order-placed"

export default defineConfig({
  // ...
  modules: [
    {
      resolve: "@medusajs/medusa/notification",
      options: {
        providers: [
          {
            resolve: "@nik0di3m/medusa-plugin-smtp-mailing/providers/smtp",
            id: "smtp-notification",
            options: {
              channels: ["email"],
              templateOverrides: {
                "order-placed": orderPlaced,
                // add more overrides as needed
              },
            },
          },
        ],
      },
    },
  ],
})
```

That's it. The plugin will use your template instead of the default one for `order-placed` emails. Any template not listed in `templateOverrides` falls back to the built-in default.

## 2. Override the base layout (branding)

If you only want to change the look (colors, logo, footer) without rewriting each template's content, override the base layout. All default templates that are NOT overridden via `templateOverrides` will use your custom layout.

### Step 1: Create a layout file

The layout function receives two arguments:
- `content` — the inner HTML body generated by the template
- `storeName` — optional store name string

```ts
// src/email-templates/my-layout.ts

export function myLayout(content: string, storeName?: string): string {
  const name = storeName || "Store"

  return `<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>${name}</title>
  <style>
    body {
      margin: 0;
      padding: 0;
      background-color: #f9fafb;
      font-family: 'Helvetica Neue', Arial, sans-serif;
    }
    .wrapper { width: 100%; padding: 40px 0; }
    .container {
      max-width: 600px;
      margin: 0 auto;
      background: #ffffff;
      border-radius: 12px;
      overflow: hidden;
    }
    .header {
      background: linear-gradient(135deg, #6366f1, #8b5cf6);
      padding: 32px;
      text-align: center;
    }
    .header img { max-height: 40px; }
    .header h1 { color: #fff; margin: 12px 0 0; font-size: 18px; }
    .content { padding: 32px; color: #1f2937; font-size: 14px; line-height: 1.7; }
    .footer {
      padding: 20px 32px;
      text-align: center;
      font-size: 12px;
      color: #9ca3af;
      border-top: 1px solid #e5e7eb;
    }
  </style>
</head>
<body>
  <div class="wrapper">
    <table class="container" cellpadding="0" cellspacing="0" width="100%">
      <tr>
        <td class="header">
          <img src="https://yourdomain.com/logo-white.png" alt="${name}" />
          <h1>${name}</h1>
        </td>
      </tr>
      <tr>
        <td class="content">${content}</td>
      </tr>
      <tr>
        <td class="footer">
          <p>&copy; ${new Date().getFullYear()} ${name}. All rights reserved.</p>
          <p><a href="https://yourdomain.com">yourdomain.com</a></p>
        </td>
      </tr>
    </table>
  </div>
</body>
</html>`
}
```

### Step 2: Register in medusa-config.ts

```ts
// medusa-config.ts
import { defineConfig } from "@medusajs/framework/utils"
import { myLayout } from "./src/email-templates/my-layout"

export default defineConfig({
  // ...
  modules: [
    {
      resolve: "@medusajs/medusa/notification",
      options: {
        providers: [
          {
            resolve: "@nik0di3m/medusa-plugin-smtp-mailing/providers/smtp",
            id: "smtp-notification",
            options: {
              channels: ["email"],
              baseLayoutOverride: myLayout,
            },
          },
        ],
      },
    },
  ],
})
```

## 3. Combine both

Override the layout for global branding AND replace specific templates:

```ts
options: {
  channels: ["email"],
  baseLayoutOverride: myLayout,
  templateOverrides: {
    "order-placed": orderPlacedCustom,
    "password-reset": passwordResetCustom,
  },
}
```

**Priority order:**
1. `templateOverrides["order-placed"]` — used as-is (your template handles its own HTML completely)
2. Default template + `baseLayoutOverride` — default content wrapped in your custom layout
3. Default template + default layout — built-in fallback

## Available templates & their data

Each template receives a `data` object. Below are the template names and the fields available in `data`:

### `order-placed`

| Field | Type | Description |
|---|---|---|
| `order_id` | `string` | Internal order ID |
| `display_id` | `string \| number` | Human-readable order number |
| `customer_name` | `string` | Full name (first + last) |
| `items` | `array` | Order items (see below) |
| `total` | `number` | Order total |
| `currency_code` | `string` | e.g. `"usd"`, `"pln"` |
| `shipping_address` | `object` | `{ address_1, city, postal_code, country_code }` |
| `store_name` | `string` | Store name |

**Item fields:** `title` or `product_title`, `quantity`, `unit_price`

### `order-completed`

| Field | Type |
|---|---|
| `order_id` | `string` |
| `display_id` | `string \| number` |
| `customer_name` | `string` |
| `store_name` | `string` |

### `order-canceled`

| Field | Type |
|---|---|
| `order_id` | `string` |
| `display_id` | `string \| number` |
| `customer_name` | `string` |
| `reason` | `string` (optional) |
| `store_name` | `string` |

### `order-fulfillment-created` (shipment notification)

Triggered by the `shipment.created` event (when a shipment is created for an order, not when a fulfillment is created). Also matched by `order-shipment-created`.

| Field | Type |
|---|---|
| `order_id` | `string` |
| `display_id` | `string \| number` |
| `customer_name` | `string` |
| `tracking_number` | `string` (optional) |
| `store_name` | `string` |

### `user-created`

| Field | Type |
|---|---|
| `email` | `string` |
| `first_name` | `string` |
| `store_name` | `string` |

### `password-reset`

| Field | Type |
|---|---|
| `email` | `string` |
| `first_name` | `string` |
| `reset_url` | `string` |
| `token` | `string` |
| `store_name` | `string` |

## Subscribed events

The plugin subscribes to the following Medusa events and sends emails automatically:

| Medusa Event | Template used | Description |
|---|---|---|
| `order.placed` | `order-placed` | Order confirmation after checkout |
| `order.completed` | `order-completed` | Order marked as complete |
| `order.canceled` | `order-canceled` | Order cancellation notice |
| `shipment.created` | `order-fulfillment-created` | Shipment notification with tracking |
| `auth.password_reset` | `password-reset` | Password reset link |
| `user.created` | `user-created` | Admin user / customer welcome |

> **Note:** The shipment email is triggered by `shipment.created` (when a shipment is created for an order), **not** by `order.fulfillment_created` (which fires earlier, when packing starts). This ensures tracking information is available in the email.

---

## Template name mapping

Templates are registered under multiple names for compatibility. You can use either name in `templateOverrides`:

| Kebab-case key | Dot-notation key |
|---|---|
| `order-placed` | `order.placed` |
| `order-completed` | `order.completed` |
| `order-canceled` | `order.canceled` |
| `order-fulfillment-created` | `order.fulfillment_created` |
| `order-shipment-created` | `order.shipment_created` |
| `user-created` | `user.created` / `customer.created` |
| `password-reset` | `auth.password_reset` |

> **Tip:** Use the kebab-case key in `templateOverrides` — it's the canonical name used by subscribers.

## Writing a template from scratch (no layout import)

You don't have to use the plugin's `baseLayout`. You can write fully standalone HTML:

```ts
// src/email-templates/order-placed.ts
import type { EmailTemplate } from "@nik0di3m/medusa-plugin-smtp-mailing/types"

export const subject: EmailTemplate["subject"] = (data) =>
  `Order #${data.display_id} confirmed`

export const html: EmailTemplate["html"] = (data) => `
<!DOCTYPE html>
<html>
<head>
  <style>
    body { font-family: sans-serif; background: #fff; padding: 40px; }
  </style>
</head>
<body>
  <img src="https://mystore.com/logo.png" alt="MyStore" height="32" />
  <h1>Order #${data.display_id}</h1>
  <p>Hi ${data.customer_name}, thanks for your purchase!</p>
  <p>Total: ${(data.currency_code || "USD").toUpperCase()} ${Number(data.total || 0).toFixed(2)}</p>
</body>
</html>
`
```

## TypeScript imports reference

```ts
// Types (for type-checking your templates)
import type { EmailTemplate, BaseLayoutFn } from "@nik0di3m/medusa-plugin-smtp-mailing/types"

// Functions (for reusing the built-in layout)
import { baseLayout, defaultBaseLayout } from "@nik0di3m/medusa-plugin-smtp-mailing/templates"
```

- `baseLayout` — the currently active layout (default or overridden via `baseLayoutOverride`)
- `defaultBaseLayout` — always returns the plugin's original built-in layout, regardless of overrides
